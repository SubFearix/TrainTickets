Система бронирования железнодорожных билетов (Railway Ticket Booking System)
Данный проект представляет собой распределенную клиент-серверную систему, предназначенную для поиска, бронирования и оплаты железнодорожных билетов.
Архитектура системы
Система построена на базе фреймворка Qt и использует следующие компоненты:
• Серверная часть (ApiServer): Многопоточный сервер, обрабатывающий подключения клиентов через QTcpServer. Управляет базой данных, сессиями пользователей и рассылкой уведомлений.
• Клиентская часть (ApiClient): Приложение с графическим интерфейсом (GUI), взаимодействующее с сервером по протоколу TCP с использованием формата JSON для обмена данными.
• База данных: Реляционная СУБД PostgreSQL для надежного хранения информации о пользователях, маршрутах, поездах и билетах.
Основные возможности
1. Управление пользователями:
    ◦ Регистрация и аутентификация с проверкой корректности Email.
    ◦ Безопасное хранение паролей с использованием хеширования Sha256 и уникальной «соли» (10 000 итераций).
    ◦ Двухфакторное подтверждение регистрации через код в Email.
    ◦ Защита от перебора: временная блокировка аккаунта на 5 минут после 5 неудачных попыток входа.
2. Поиск и бронирование:
    ◦ Поиск поездов по станциям отправления/прибытия и дате.
    ◦ Интерактивный выбор мест в вагонах различных типов (Плацкарт, Купе, Сидячий).
    ◦ Механизм тайм-аута бронирования: на оплату билета отводится 15 минут, после чего бронь аннулируется.
3. Оплата и получение билета:
    ◦ Проведение оплаты через клиентский интерфейс.
    ◦ Автоматическая генерация электронного билета в формате PDF.
    ◦ Отправка билета пользователю на электронную почту через встроенный SMTP-клиент.
4. Личный кабинет:
    ◦ Просмотр истории заказов (активные и завершенные билеты).
    ◦ Возможность отмены бронирования.
    ◦ Редактирование профиля и смена пароля.
Технологический стек
• Язык программирования: C++17.
• Библиотеки Qt: network (TCP сокеты), sql (PostgreSQL драйвер), gui/widgets (интерфейс), cryptographichash (безопасность).
• Формат данных: JSON (библиотека QJsonDocument).
• Дополнительно: QPdfWriter для формирования билетов.
Структура проекта
• ApiClient — синглтон для управления сетевым состоянием на стороне клиента.
• ClientHandler — класс на стороне сервера, отвечающий за логику обработки команд конкретного пользователя.
• Database — фасад для работы с SQL-запросами и инициализации таблиц базы данных.
• MainWindow — центральное окно приложения, использующее QStackedWidget для навигации между экранами (Вход, Меню, Выбор мест, Оплата).
• PdfGenerator — модуль для создания визуального представления билета.
Настройка и запуск
1. Установите параметры подключения к БД в файле config/database.conf.
2. Настройте параметры почтового сервера (SMTP) в config/email.conf для корректной отправки билетов.
3. Запустите серверную часть (ApiServer).
4. Запустите клиентское приложение. По умолчанию клиент подключается к 127.0.0.1:8080.
